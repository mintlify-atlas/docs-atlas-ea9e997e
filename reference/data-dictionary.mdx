## Account & Billing

These fields come from the nested `account` object (and its nested `Charges` sub-object) in the raw JSON.

| Field | Type | Description | Values |
|-------|------|-------------|--------|
| `Contract` | string | The customer's contract commitment type. The most impactful categorical predictor of churn. Spanish alias: `Tipo_Contrato`. | `"Month-to-month"`, `"One year"`, `"Two year"` |
| `PaperlessBilling` | string | Whether the customer has opted for paperless (digital) billing. | `"Yes"`, `"No"` |
| `PaymentMethod` | string | The customer's payment method. Electronic check is the most common (~33.6% of customers). | `"Electronic check"`, `"Mailed check"`, `"Bank transfer (automatic)"`, `"Credit card (automatic)"` |
| `Charges.Monthly` | float | Total monthly charges across all active services. Higher charges correlate positively with churn (r = +0.19). Spanish alias: `Cargo_Mensual`. | Float â‰Ą 0.0 |
| `Charges.Total` | float | Total amount the customer has paid over their entire tenure. **Stored as a string in the raw JSON** and must be cast to float during cleaning. Empty strings (new customers with no charges yet) are converted to `0`. Spanish alias: `Cargo_Total`. | Float â‰Ą 0.0 (after conversion) |

<Warning>
  `Charges.Total` is stored as a string type (`object`) in the raw JSON. During the ETL pipeline, it is converted with `pd.to_numeric(..., errors='coerce').fillna(0)`. Failing to handle this causes silent NaN propagation in downstream calculations.
</Warning>

## Engineered Features

The following columns do not exist in the raw data. They are created during the transformation and feature engineering steps of the notebook pipeline.

| Field | Type | Formula / Origin | Description |
|-------|------|-----------------|-------------|
| `Charges.Daily` | float | `Charges.Monthly / 30` | Daily cost approximation, rounded to 2 decimal places. Created to enable finer-grained analysis of billing pressure. In the notebook this column is named `Cuentas_Diarias` (Spanish alias: `Cargo_Diario`). |
| `Churn_binary` | integer | `1` if `Churn == "Yes"`, else `0` | Binary encoding of the target variable. Created during the standardization step to enable Pearson correlation calculations and ML model training. In the notebook this is part of the `Abandono` column after the binary mapping step. |

```python
# Feature engineering: Charges.Daily
df_limpio['Cuentas_Diarias'] = (df_limpio['Charges.Monthly'] / 30).round(2)

# Feature engineering: Churn_binary (via binary mapping)
mapeo_binario = {'Yes': 1, 'No': 0}
df_telecom_es['Abandono'] = df_telecom_es['Abandono'].map(mapeo_binario)
```
